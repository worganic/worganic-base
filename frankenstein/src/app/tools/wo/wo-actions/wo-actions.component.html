2 Sélectionner et lancer les prompts

check_box

Cochez une ou plusieurs actions dans le tableau ci-dessous (les colonnes Nom et Prompt sont les plus importantes).

visibility

Cliquez sur Voir les prompts sélectionnés — un popup s'ouvre avec la liste et le bouton Lancer les prompts.

settings

Le fournisseur IA et le modèle utilisés sont ceux sélectionnés dans la page Config du site. Vous pouvez les changer avant de lancer.

rocket_launch

Cliquez sur Lancer les prompts : la page live s'ouvre automatiquement dans un nouvel onglet.

3 Ce qui se passe pour chaque prompt

Branche git créée — une branche action/<id>-<nom> est créée depuis main.

IA exécutée — le prompt est envoyé à la CLI IA (Claude ou Gemini) qui modifie les fichiers du projet directement.

Chat disponible — si l'IA pose une question ou attend une réponse, utilisez le champ de chat en bas de la page live pour lui répondre.

Commit + Push — les fichiers modifiés sont commités et la branche est poussée vers le dépôt distant (si configuré).

Rapport enregistré — la liste des fichiers modifiés, le nom de branche et le hash de commit sont sauvegardés sur l'action. Cliquez sur Rapport → dans le tableau pour les consulter.

4 Page live — suivi et reprise

open_in_new

Accès direct

La page live s'ouvre automatiquement au lancement. URL : /agent/live/<runId>

history

Reprise après fermeture

Si la page a été fermée, cliquez sur Revoir dans le bandeau en haut de cette section. Tout l'historique est rejoué depuis le serveur.

chat

Chat bidirectionnel

Pendant l'exécution, tapez dans le champ en bas de la page live et appuyez sur Entrée pour envoyer une réponse à l'IA.

stop_circle

Arrêt propre

Cliquez sur Arrêter (bandeau ou page live) pour stopper le batch après le prompt en cours. La branche en cours reste intacte.
