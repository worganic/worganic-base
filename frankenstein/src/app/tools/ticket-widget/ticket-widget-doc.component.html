Composant <wo-ticket-widget>

Widget standalone de signalement de bugs et d'améliorations. Capture automatiquement une copie d'écran via html2canvas, permet de l'annoter au trait, puis envoie le ticket enrichi à l'API. Aucun @Input() ni @Output() — le composant est entièrement autonome.

Annotation Canvas

Le système d'annotation utilise deux éléments <canvas> superposés : #bgCanvas (fond, lecture seule) et #drawCanvas (calque de dessin transparent).

layers Architecture des calques

photo
#bgCanvas — Calque fond (position: static)

Reçoit drawImage() du screenshot au chargement. Lecture seule — aucun événement souris. Largeur fixée à 1000px, hauteur calculée proportionnellement.

brush
#drawCanvas — Calque dessin (position: absolute, top: 0, left: 0)

Superposé sur bgCanvas. Reçoit tous les événements souris/touch. Fond transparent. La fusion est faite à la soumission via getMergedScreenshot().

API & Backend

Endpoint requis et structure du payload envoyé.

cloud_upload Endpoint — POST /api/tickets

POST /api/tickets Nécessite le header Authorization
Payload JSON envoyé
{
                "title":       "string (requis)",
                "description": "string (requis)",
                "url":         "string",
                "type":        "bug | amelioration | ui | contenu | performance",
                "priority":    "critique | haute | normale | basse",
                "screenshot":  "string | null  ← base64 PNG"
              }
Réponse succès
HTTP 200 / 201
                { "id": "...", "status": "ok" }
                → Appel de close() automatique
                → Retour en phase "idle"
                Erreur :
                { "error": "message" }
              → Affichage dans la zone erreur

lock Authentification

Le composant utilise AuthService.getAuthHeaders() pour ajouter le header Authorization à la requête. Le token est lu depuis le localStorage clé frankenstein_token. 

// Dans ticket-widget.component.ts
            await this.http.post(`${API}/api/tickets`, payload, {
            headers: this.auth.getAuthHeaders()
          }).toPromise();

storage Stockage backend

Les tickets sont persistés dans data/tickets.json via server/server-data.js. La liste est consultable dans la section Tickets de l'interface admin à l'adresse /tickets.

Intégration

Le widget ne nécessite qu'un import et une balise HTML — aucun @Input() ni @Output().

add_circle 1. Import 

import { TicketWidgetComponent } from '../../components/ticket-widget/ticket-widget.component';
            @Component({
            imports: [TicketWidgetComponent, ...]
          })

html 2. Template — usage complet 

<!-- Placer en bas de la page hôte, hors des conteneurs overflow-hidden -->
          <wo-ticket-widget></wo-ticket-widget>

Le bouton flottant est positionné en fixed bottom-6 left-6 z-[500]. Aucune configuration supplémentaire requise.

integration_instructions 3. Exemple — app.component.html 

<router-outlet></router-outlet>
            <!-- Widget visible sur toutes les pages de l'app -->
          <wo-ticket-widget></wo-ticket-widget>

tips_and_updates Bonnes pratiques

  • check Placer la balise à la racine du composant hôte, jamais dans un conteneur avec overflow: hidden
  • check Le z-index 500 garantit que le widget s'affiche au-dessus des modals standards
  • warning html2canvas peut échouer sur des iframes ou des éléments cross-origin — le composant gère le cas null (affiche "capture non disponible")
  • warning La capture est faite à l'ouverture du widget (avant l'affichage du formulaire) pour éviter de capturer le formulaire lui-même
